KEY TOPICS (bullet summary)
--------------------------------------------------------------------------------
  - Operator-experience problem: ratchet apply --pr --totp is unusable
    months after last use
  - Telegram bot is *transport only*: it never sees the TOTP seed and
    never grants authority
  - Out-of-band approval still ends in the agent’s local verifier
  - HMAC X-Hub-Signature-256 verification on the inbound GitHub webhook
    is mandatory before any Telegram notification fires
  - Telegram outage must be fail-closed; CLI fallback always available
  - Audit receipt records approved_via: telegram for forensics
  - Hardening inconsistency: SSH masked on N95 but reachable on N305
  - The systemd trap: stopping ssh.service while ssh.socket survives
    → socket-activation wakes the service back up
  - Fix is IaC, not memory: idempotent Bash script applied to every
    proxmox-host class via the agent’s pct exec adapter
  - Post-apply self-check verifies systemctl is-masked for both units
