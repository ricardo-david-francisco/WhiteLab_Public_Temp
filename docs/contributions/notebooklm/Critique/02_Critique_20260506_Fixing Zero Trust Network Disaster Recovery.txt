KEY TOPICS (bullet summary)
--------------------------------------------------------------------------------
  - Failure mode: at 03:00 the operator improvises on the OPNsense
    physical console, then walks away — drift is born
  - Either the next reconciliation overwrites the emergency fix and
    resurrects the outage, or the agent halts on drift detection
  - Break-glass primitive: locally signed payload, posted directly to
    the agent over Tailscale, no GitHub round-trip
  - DNS may be the failing service — magic-DNS unavailable → use the
    agent’s static Tailscale IP, not its hostname
  - Cryptographic boundary unchanged: Gate 6 (TOTP) and Gate 7
    (snapshot) still apply
  - sync-drift produces a retroactive recovery/<date> branch that
    captures the manual change as code; PR → review → merge
  - Receipt format identical to normal apply path; downstream tooling
    needs no special case
  - Both subcommands belong in docs/runbooks/break-glass.md and must
    be rehearsed at least once before production reliance
