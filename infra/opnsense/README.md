# OPNsense — anonymized exports + deltas

* `exports/` — sanitized snapshots of `config.xml` produced by
  `tools/anonymizer/anonymize.py --format opnsense_xml`. Generated by
  the Fortress Agent's `pull` adapter; never authored by a human.
* `deltas/` — Jinja-templated change fragments authored by humans (or
  Copilot). Naming: `YYYY-MM-DD-short-slug.xml.j2`. Each delta declares
  the XPath roots it intends to mutate; the apply pipeline fails if any
  delta touches a root not on its declared list.
* `schemas/` — XSD or relaxed-NG schemas used by `xmllint --noout` in
  CI to catch syntactically-broken edits before they ship.

The agent's apply flow:

```text
exports/config.anonymized.xml  +  deltas/*.xml.j2
        │
        ▼ rehydrate (agent-only)
        ▼ render Jinja
        ▼ xmllint --schema
        ▼ POST /api/core/backup/upload  (config.xml)
        ▼ POST /api/firewall/filter/apply
```

Pull flow:

```text
GET /api/core/backup/download/this  →  raw config.xml
        ▼ anonymize --format opnsense_xml
        ▼ diff against exports/config.anonymized.xml
        ▼ if changed: open chore(opnsense): drift PR
```
